### S2. Sequence Diagram — OAuth2 PKCE Flow

Multi-participant sequence with notes, spans, and groups.

```d2
shape: sequence_diagram

browser: Browser
auth: Auth Server
api: API Server
db: Database

browser -> auth: 1. GET /authorize?code_challenge=xyz
auth -> browser: 2. Redirect to login page
browser -> auth: 3. POST /login (credentials)
auth -> auth: 4. Validate credentials

auth -> browser: 5. Redirect with authorization code

browser -> auth: 6. POST /token (code + code_verifier)
auth -> auth: 7. Verify code_challenge matches SHA256(code_verifier)

auth -> browser: 8. Return access_token + refresh_token

browser -> api: 9. GET /api/data (Bearer token)
api -> auth: 10. Validate JWT signature
auth -> api: 11. Token valid
api -> db: 12. SELECT data
db -> api: 13. Result set
api -> browser: 14. 200 OK (data)

browser -> auth: 15. POST /token (refresh_token)
auth -> auth: 16. Rotate refresh token
auth -> browser: 17. New access_token + refresh_token
```
